# Google Sheets 구성

이 프로젝트의 Google Sheets는 운영자가 직접 보는 업무 화면이자, Apps Script가 상태를 갱신하는 데이터 저장소입니다.

운영 시트:

https://docs.google.com/spreadsheets/d/1ZNPijMHvyZXXKVdr72q4DEEi8YJxpY0RU_IT_Y0xS14/edit

## 편성표

프로그램별 최종 입고 상태를 확인하는 메인 시트입니다.

주요 열:

- `채널`: 방송 채널
- `프로그램`: 편성표 기준 프로그램명
- `회차`: 파일 수 계산 기준
- `파일 입고시간`: 모든 파일이 들어왔을 때 마지막 발생시간을 `HH:mm` 형식으로 기록

회차 입력 규칙:

- `23`: 단일 회차, 파일 1개
- `23-24`: 범위 회차, 파일 2개
- `181~`: 끝 회차 미확정, 완료 판단 제외 또는 확인필요 관리

## 입고로그

텔레그램에서 들어온 메시지와 처리 결과를 보관합니다.

로그에는 파일명, 경로, 발생시간, 매칭 방식, 처리 상태, 텔레그램 메시지 키 등이 저장됩니다. 같은 메시지 ID 또는 같은 프로그램키+파일명 조합은 중복 처리하지 않는 방향으로 설계되어 있습니다.

## 미매칭검수

자동으로 확정하기 어려운 메시지를 운영자가 직접 연결하는 시트입니다.

예시:

- 편성표 프로그램명: `스브스뉴스`
- 텔레그램 경로명: `SBS_NEWS`

이 경우 자동으로 완료 처리하지 않고 미매칭검수에 모읍니다. 운영자가 편성표 항목을 선택하면 해당 파일도 즉시 입고 카운트에 반영됩니다.

## 대시보드

전체 진행 상태를 요약합니다.

- 전체 프로그램 수
- 완료
- 입고중
- 미입고
- 확인필요
- 마지막 갱신 시각

## 테스트메시지

텔레그램 그룹방에 복사해서 보낼 수 있는 테스트 메시지를 관리합니다.

현재 테스트 방향:

- 정상 매칭: 경로의 프로그램명이 편성표 프로그램명과 일치
- 미매칭검수: 텔레그램 경로명이 편성표 프로그램명과 달라 검수 시트로 이동

## Apps Script 설치

`apps-script/Code.gs` 내용을 Google Apps Script의 `Code.gs`에 전체 붙여넣은 뒤 웹 앱으로 배포합니다.

배포 설정:

- 실행 사용자: 나
- 액세스 권한: 모든 사용자

Telegram webhook은 Cloudflare Worker 주소로 연결하고, Worker가 Apps Script Web App URL로 요청을 전달합니다.
